API REST - Gonzalo Gonzalez

Endpoints Users ▶

GET /api/users ▶
Obtiene la lista de todos los usuarios.

Respuesta exitosa (200 OK)


    [
        {
            "id": 1,
            "name": "Gonzalo",
            "last_name:" "Gonzalez"
            "username": "gonzalo@example.com"
        }
    ]

Errores comunes

  • 500 Internal Server Error: Problema en el servidor.
GET /api/users/:id ▶
Obtiene un usuario por su ID.

Parámetros

  • id (path): ID del usuario.

Respuesta exitosa (200 OK)

    {
        "id": 1,
        "name": "Gonzalo",
        "last_name:" "Gonzalez"
        "username": "gonzalo@example.com"
    }

Errores comunes

  • 404 Not Found: Usuario no encontrado.
  • 400 Bad Request: ID inválido.
POST /api/users ▶
Crea un nuevo usuario.

Body (JSON)

    {
        "name": "Nuevo",
        "last_name": "Usuario",
        "username": "nuevo.usuario@example.com",
        "password": "secretPassword"
    }

Respuesta exitosa (201 Created)

    {
        "id": 2,
        "name": "Nuevo",
        "last_name": "Usuario",
        "username": "nuevo.usuario@example.com"
    }

Errores comunes

  • 400 Bad Request: Datos faltantes o inválidos.
  • 409 Conflict: Email ya registrado.
PUT /api/users/:id ▶
Actualiza un usuario existente.

Parámetros

  • id (path): ID del usuario a actualizar.

Body (JSON)

    {
        "name": "Gonzalo",
        "last_name": "Actualizado",
        "username": "gonzalo.actualizado@example.com",
        "password": "secretPasswordUpd"
    }

Respuesta exitosa (200 OK)

    {
        "id": 1,
        "name": "Gonzalo",
        "last_name": "Actualizado",
        "username": "gonzalo.actualizado@example.com"
    }

Errores comunes

  • 400 Bad Request: Datos inválidos.
  • 404 Not Found: Usuario no encontrado.
DELETE /api/users/:id ▶
Elimina un usuario por su ID.

Parámetros

  • id (path): ID del usuario a eliminar.

Respuesta exitosa

        "message": "Usuario eliminado correctamente",
